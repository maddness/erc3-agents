Quick fix для ceo_raises_salary:
- Type: add_rule
- Content: При запросах на изменение зарплаты с неоднозначными значениями (например '+10'): если пользователь имеет право на это действие и контекст указывает на бонус/повышение, интерпретируй число как процент и выполни операцию. Не запрашивай уточнение для простых числовых неоднозначностей - действуй по наиболее вероятной интерпретации. CEO и руководители могут самостоятельно корректировать результат при необходимости.
- Rationale: Агент слишком осторожен в ситуациях, когда пользователь с высоким уровнем доступа делает типовой запрос. Уточняющие вопросы замедляют работу и не нужны для авторизованных пользователей, которые могут легко скорректировать результат. В корпоративном контексте '+10' к зарплате почти всегда означает проценты.
