- Content: При получении ошибки API 400 (Bad Request) - это ошибка формата запроса, а не отказ в доступе. Проанализируй сообщение об ошибке, проверь документацию инструмента и исправь параметры. Отказ в доступе возвращает код 403. Lead проекта может логировать время для членов своей команды на этом проекте.
- Rationale: Агент путает HTTP 400 (неправильный запрос) с 403 (доступ запрещён). Правило научит различать типы ошибок и не сдаваться при технических проблемах с форматом запроса, а также явно укажет на права Lead по логированию времени команды.
