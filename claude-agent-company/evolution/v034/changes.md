Quick fix для add_time_entry_lead:
- Type: add_rule
- Content: При логировании времени по умолчанию employee должен быть текущий пользователь (из whoami). Фразы типа 'for [person]' или 'on [project]' описывают контекст работы, а НЕ указывают другого исполнителя. Логировать время за ДРУГОГО сотрудника можно ТОЛЬКО при явном указании формата 'log time AS [employee]' или 'log [employee]'s time' или прямом вопросе 'запиши время [имя сотрудника]'. При любой неоднозначности - уточнить у пользователя.
- Rationale: Правило устраняет двусмысленность интерпретации 'for X' - это контекст работы, а не указание исполнителя. Это предотвратит неправильное присвоение времени другим сотрудникам.
