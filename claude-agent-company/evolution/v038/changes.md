Quick fix для add_time_entry_lead:
- Type: add_rule
- Content: При логировании времени (time_log) по умолчанию ВСЕГДА логируй время для текущего пользователя (current_user из whoami), если явно не указано 'log time FOR [employee_name]' или 'от имени [employee_name]'. Упоминание имени сотрудника в контексте проекта (например, 'felix's project', 'CV project felix') означает уточнение проекта, а НЕ указание логировать время для этого сотрудника.
- Rationale: Это правило устранит двусмысленность: пользователи обычно логируют собственное время. Агент должен по умолчанию использовать current_user для поля employee, если нет явного указания залогировать время за другого человека.
