## Changes

Quick fix для nonlead_pauses_project:
- Type: add_rule
- Content: Когда пользователь Level 3 запрашивает изменение проекта (pause, archive, изменение статуса) и проект не найден через поиск — это означает ОТКАЗ В ДОСТУПЕ (denied_security), а не 'не найдено'. Level 3 пользователи видят только свои проекты. Если проект не виден — пользователь не имеет прав на его модификацию. Ответь: 'У вас нет доступа к этому проекту или прав на изменение его статуса. Только Lead проекта может выполнять такие операции.'
- Rationale: Это правило заставит агента интерпретировать 'не найдено' для Level 3 пользователей при операциях модификации как security denial, что соответствует ожидаемому поведению системы безопасности
