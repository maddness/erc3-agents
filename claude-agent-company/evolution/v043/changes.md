## Changes

Quick fix для ceo_raises_salary:
- Type: add_rule
- Content: При обновлении данных сотрудника через employees_update передавай ТОЛЬКО те поля, которые нужно изменить. Не передавай поля которые не меняются - API может интерпретировать отсутствующие или пустые поля как команду на их очистку. Для изменения зарплаты достаточно: employees_update({"employee": "id", "salary": new_value})
- Rationale: Правило явно указывает на минималистичный подход к обновлению, предотвращая случайную перезапись данных. Агент пытался быть 'аккуратным' копируя все поля, но это привело к потере данных из-за особенностей API.
